# Portfolio Website Build Prompt for AI Engineer

## Project Overview
Build a modern, professional portfolio website for an AI Engineer that showcases technical expertise, projects, and professional experience. The site should have a sophisticated, mechanical aesthetic with smooth animations and interactive elements.

## Data Source
- **GitHub Repository**: https://github.com/Alli-Ekundayo
- Extract and display: repositories, project descriptions, technologies used, contribution statistics, and pinned projects
- Fetch profile information including bio, location, and links

## Design Requirements

### Hero Section - Animated Mechanical Blueprint
**Primary Feature**: An animated exploded blueprint diagram (choose one):
- Option 1: Mechanical watch (showing gears, springs, escapement mechanism)
- Option 2: Jet engine (turbine blades, combustion chamber, compressor stages)
- Option 3: Other intricate mechanical device (clockwork mechanism, transmission system)

**Animation Specifications**:
- Start with assembled view, then smoothly explode/separate components
- Each component should float outward from center with subtle rotation
- Display thin connecting lines (blueprint style) between related parts
- Components should be labeled with technical annotations (like engineering drawings)
- Use monochromatic color scheme: blues/teals on dark background OR sepia/charcoal on light background
- Animation should loop: explode → pause → reassemble → pause → repeat
- Smooth easing functions (ease-in-out) for professional feel
- Duration: 8-12 seconds per complete cycle
- Implement using SVG or Canvas for crisp rendering
- Should be responsive and scale on different screen sizes

### Color Scheme & Typography
- **Primary Colors**: 
  - Background: Deep charcoal (#1a1a1a) or dark navy (#0f1419)
  - Accent: Electric blue (#2d8cf0) or cyan (#00d9ff)
  - Text: Light gray (#dedede) with white (#ffffff) for emphasis
- **Typography**:
  - Headings: Modern sans-serif (Inter, Poppins, or Space Grotesk)
  - Body: Clean, readable sans-serif (System UI, Segoe UI, or Roboto)
  - Code snippets: Monospace (JetBrains Mono, Fira Code)

### Layout Sections

1. **Hero Section**
   - Animated mechanical blueprint as background
   - Name and title overlay ("AI Engineer")
   - Brief tagline (1-2 sentences about specialization)
   - CTA buttons using the provided button design

2. **About Section**
   - Professional headshot or avatar
   - Bio paragraph highlighting AI/ML expertise
   - Key skills with visual indicators (progress bars or icon badges)
   - Specializations: NLP, Computer Vision, Deep Learning, MLOps, etc.

3. **Projects Showcase**
   - Grid layout of GitHub repositories (3 columns on desktop, responsive)
   - Each project card should display:
     - Project name and description
     - Primary language and tech stack icons
     - GitHub stars/forks count
     - "View Project" button (using provided design)
     - "Live Demo" button if applicable (using provided design)
   - Filter by technology tags
   - Sort by: Recent, Most Starred, Language

4. **Skills & Technologies**
   - Visual grid of technology logos/icons
   - Categories: Languages, Frameworks, Tools, Cloud Platforms
   - Hover effects showing proficiency level

5. **Experience/Timeline** (if data available)
   - Professional experience
   - Education
   - Certifications
   - Publications or notable contributions

6. **Contact Section**
   - Email contact form or mailto link
   - Social links: GitHub, LinkedIn, Twitter, etc.
   - "Download Resume" button (using provided design)
   - Location information if provided

## Technical Implementation

### Tech Stack Recommendation:
- **Frontend**: React with TypeScript or Next.js (for SEO)
- **Styling**: TailwindCSS + custom CSS for button component
- **Animations**: Framer Motion or GSAP for mechanical blueprint
- **API Integration**: GitHub REST API or GraphQL API
- **Deployment**: Vercel, Netlify, or GitHub Pages

### Key Features:
1. **Responsive Design**: Mobile-first, works perfectly on tablets and desktops
2. **Performance**: Fast load times, lazy loading for images
3. **SEO Optimized**: Meta tags, OpenGraph tags, schema markup
4. **Accessibility**: WCAG 2.1 AA compliant, keyboard navigation
5. **Dark Mode**: Default dark theme (optional toggle to light mode)
6. **Smooth Scrolling**: Animated transitions between sections
7. **Loading States**: Skeleton screens while fetching GitHub data

### GitHub API Integration:
```javascript
// Fetch user data
GET https://api.github.com/users/Alli-Ekundayo

// Fetch repositories
GET https://api.github.com/users/Alli-Ekundayo/repos?sort=updated&per_page=20

// Fetch specific repo details
GET https://api.github.com/repos/Alli-Ekundayo/{repo-name}
```
